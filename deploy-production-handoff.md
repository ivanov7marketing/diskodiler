## Какие проблемы нашли и как исправили

### 1. После переключения nginx сайт отдавал `500`

Причина оказалась составной:

- `storage/*` и `bootstrap/cache` были заполнены файлами от `root`, а web-runtime работал не с теми правами.
- production `.env` имел права `600` и владельца `deploy`, поэтому web-процесс не мог его читать.
- из-за этого Laravel видел пустой `APP_KEY` и падал с `MissingAppKeyException`.
- app-контейнер был уже создан раньше, поэтому даже после исправления `.env` продолжал жить со старыми env-переменными из `env_file`.

Что сделали:

- привели writable-директории к корректным правам;
- открыли чтение `.env` для runtime-группы контейнера;
- пересобрали Laravel cache под корректным пользователем;
- пересоздали `app` контейнер, чтобы он перечитал env;
- повторно прогнали cache-команды.

### 2. Вход в админку не работал

Причины:

- ранее выданный пароль содержал спецсимволы и, вероятно, был искажен при серверной команде;
- у пользователя `admin@diskodiler.shop` поле `email_verified_at` было пустым, а `canAccessPanel()` требует верифицированного admin.

Что сделали:

- сбросили пароль на простой и проверяемый;
- выставили `is_admin = true`;
- заполнили `email_verified_at`;
- проверили, что хэш пароля совпадает и запись пользователя корректная.

## Что можно дополнительно потестировать вручную

Ниже чеклист, который полезно пройти в браузере и частично на сервере, чтобы убедиться, что сайт действительно работает как надо.

### 1. Главная и навигация

- открыть главную страницу и проверить, что она рендерится без визуальных ошибок
- проверить меню в desktop и mobile
- проверить переходы:
  - главная -> каталог
  - главная -> услуги
  - главная -> доставка
  - главная -> о компании
- проверить, что логотип возвращает на главную

### 2. Каталог

- открыть `/diski`
- проверить, что видны бренды и товары
- открыть `/diski/bmw`
- открыть карточку товара
- проверить, что:
  - показываются изображения
  - есть цена или price label
  - есть связанные товары
  - нет битых картинок
- пройти по нескольким брендам и моделям, если они доступны в выдаче

### 3. Формы заявок

Проверить несколько сценариев:

- форма VIN на главной
- форма VIN в каталоге
- форма на карточке товара
- форма на странице услуг

Для каждой формы проверить:

- успешную отправку
- отображение сообщения об успехе
- появление заявки в админке
- сохранение нужных полей:
  - имя
  - телефон
  - способ связи
  - сообщение / VIN
  - `source_page`
  - `goal`

После тестов удалить мусорные тестовые заявки.

### 4. Админка Filament

- зайти в `/admin/login`
- войти под `admin@diskodiler.shop`
- проверить доступность разделов:
  - Dashboard
  - Brands
  - Vehicle Models
  - Products
  - Leads
- открыть список лидов и убедиться, что там появляются новые заявки
- открыть товар и проверить, что данные читаются без ошибок

### 5. SEO и технические страницы

- открыть `/robots.txt`
- убедиться, что содержимое соответствует текущему режиму индексации
- открыть `/sitemap.xml`
- убедиться, что sitemap отдается как XML
- проверить meta title / description на:
  - главной
  - каталоге
  - бренде
  - карточке товара

### 6. Anti-indexing

Сейчас для тестового домена была настроена антииндексация.

Нужно отдельно проверить:

- есть ли `<meta name="robots" content="noindex,nofollow">`
- соответствует ли этому содержимое `/robots.txt`

Если сайт готовится к реальному публичному запуску, позже нужно отдельно решить, когда снимать anti-indexing.

### 7. Проверка после логина в админку

- создать тестовый бренд или отредактировать существующий
- сохранить изменения
- убедиться, что Filament не падает на сохранении
- проверить загрузку изображений товара, если планируется работа с media из админки

### 8. Проверка серверной части

На сервере полезно периодически проверить:

- `docker compose -f docker-compose.prod.yml ps`
- последние логи `app`
- последние логи `db`
- что контейнеры не рестартуют циклически
- что Laravel пишет логи без permission issues

## Полезные production-команды

Из `/opt/diskodiler/laravel`:

```bash
docker compose -f docker-compose.prod.yml ps
docker compose -f docker-compose.prod.yml logs --tail=200 app
docker compose -f docker-compose.prod.yml logs --tail=200 db
docker compose -f docker-compose.prod.yml exec -T app php artisan optimize:clear
docker compose -f docker-compose.prod.yml exec -T app php artisan config:cache
docker compose -f docker-compose.prod.yml exec -T app php artisan route:cache
docker compose -f docker-compose.prod.yml exec -T app php artisan view:cache
docker compose -f docker-compose.prod.yml exec -T db psql -U diskodiler -d diskodiler
```
